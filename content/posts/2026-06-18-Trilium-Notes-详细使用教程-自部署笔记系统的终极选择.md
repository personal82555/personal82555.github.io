    去年有个朋友找我推荐笔记软件，说Notion太慢、Obsidian要折腾插件、飞书文档又太贵。我给他安利了Trilium Notes，用了一周他跟我说：「这是唯一一个让我感觉是在搭建自己的知识库，而不是在填表格的笔记软件。」 

Trilium Notes（简称Trilium）是一款**开源、自托管的层级笔记应用** ，支持树形结构、富文本编辑、关系图谱、版本历史、加密笔记、发布分享等功能。跟市面上大多数「文件夹+标签」的笔记方案不同，Trilium用的是**无限级树形结构** ——每个笔记都可以有无限个子笔记，而且一个笔记可以出现在多个父节点下（多父级关联），非常适合知识管理场景。 

##  Trilium 的核心特色 

**1\. 树形笔记结构**

左侧是一棵笔记树，你可以自由拖拽调整层级。一个「知识库」根节点下，可以按类别建「技术笔记」「写作输出」「项目管理」等二级目录，每个目录下再细分。笔记之间还能通过「关系链接」建立引用，不像文件夹系统那样死板。 

**2\. 自托管 & 数据全在你手里**

Trilium 的数据存在你自己的服务器上（SQLite 数据库），可以部署在 NAS、VPS、树莓派甚至本地电脑。所有笔记都是纯 HTML 格式存储，导出为 Markdown、PDF 都很方便。不存在「某天服务商倒闭笔记全丢」的风险。 

**3\. 开源 & 社区活跃**

GitHub 上 30K+ Star，原生支持 REST API（ETAPI），还有活跃的社区分支 TriliumNext 增加了更多功能。完全免费，无任何付费墙。 

##  界面总览 

![](https://img.88531.cn/i/2026/06/21/6a36bd3965f9d.png)

Trilium 的界面分为三大区域： 

  * 📂 **左侧树形目录** ：按层级展示所有笔记，支持拖拽排序、拖拽移动
  * 🖊️ **中间编辑区** ：CKEditor 5 所见即所得编辑器，支持表格、图片、代码块
  * 🔧 **顶部工具栏** ：笔记操作、搜索、发布、历史版本、关系图谱等高级功能

##  如何部署 Trilium？ 

部署方式非常灵活，推荐以下三种： 

**方式** | **命令** | **适合场景**  
---|---|---  
Docker | `docker run -d -p 8080:8080 -v ~/trilium-data:/home/trilium-data zadam/trilium` | NAS 用户、不想装 Node.js  
直接运行 | 下载 release → 解压 → `./trilium.sh` | VPS/本地服务器直跑  
源码编译 | `git clone && pnpm install && pnpm server:build` | 需要魔改源码的高级用户  
  
首次启动后，浏览器打开 `http://你的IP:8080`，设置密码即可开始使用。Trilium 默认开启 HTTPS 访问提示，如果用反向代理配合 HTTPS，体验更佳。 

##  日常使用技巧 

**🔍 全局搜索**

顶部搜索框支持全文检索，输入关键词立刻找到所有匹配的笔记。搜索速度非常快，因为数据存储在本地 SQLite 数据库中。 

**🔗 关系图谱（Relation Map）**

你可以在笔记之间建立「关系链接」，比如「A笔记是B笔记的前置知识」「C笔记参考了D笔记」。点击关系图谱按钮，这些链接会以图形化方式展示，让你一眼看清知识之间的关系网。 

**📤 分享与发布**

Trilium 可以为单个笔记生成分享链接（`/share/{noteId}`），无需登录即可查看。对于自媒体创作者，还可以通过 ETAPI 将文章发布到 WordPress、微信公众号、CSDN 等多个平台，实现「写一次，到处发布」。这正是我目前的工作流：Trilium 写文章 → ETAPI 自动发布 → 多平台同步。 

**🔄 版本历史**

每次保存时自动记录版本，你可以随时回滚到任意历史版本。写出烂稿了也不怕——Ctrl+Z 只能撤销最近操作，但版本历史能让你找回三天前的版本。

##  ETAPI：自动化是最大亮点 

Trilium 提供了完整 REST API（也就是 ETAPI），我每天用它来做自动化发布。一条命令就能创建笔记、上传图片、设置分享标签，全部通过 HTTP 请求完成，不用打开浏览器。 

比如搭配 AI 生成文章，然后自动发布到 Trilium 的日期目录下，再自动同步到 WordPress 和公众号——整个过程全自动，每天定时跑。这是 Trilium 对比 Obsidian、Notion 最具差异化的优势：**可以被编程控制** 。 

ETAPI 常用操作示例： 

  * ⚡ `GET /etapi/notes/{id}` — 获取笔记信息
  * ⚡ `POST /etapi/create-note` — 创建新笔记（支持 HTML 内容）
  * ⚡ `PUT /etapi/notes/{id}/content` — 更新笔记内容
  * ⚡ `POST /etapi/attributes` — 添加标签（如分享标签）

##  总结 

Trilium Notes 适合那些对数据主权有要求的用户——不想把笔记存在别人的服务器上，又想要一个功能完整的笔记系统。它的树形结构让知识管理更直观，ETAPI 接口让自动化成为可能，内置分享功能方便对外发布。 

当然它也有不足之处：手机端只有 Web App 没有原生 App，对 Markdown 的支持需要插件或手动转换，多用户协作方面比较弱。但对于**个人知识管理和自媒体创作** 来说，Trilium 是我用过最顺手的方案。 

如果你正在找一个能自己掌控数据、支持自动化、写笔记不是为了填表格而是真正搭建知识库的工具，Trilium 值得一试。
